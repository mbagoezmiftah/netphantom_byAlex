# ARP 欺骗使用说明

## 工作原理

ARP 欺骗通过发送伪造的 ARP 响应包,让目标主机误以为攻击者的 MAC 地址是网关的 MAC 地址,从而将发往网关的流量发送到攻击者机器。

## 使用步骤

### 1. 扫描网络
- 在"主机扫描"页面扫描网段
- 找到要欺骗的目标主机

### 2. 启动欺骗
- 双击目标主机,自动跳转到"ARP 欺骗"页面
- 点击"自动检测"获取网关 IP
- 点击"开始欺骗"

### 3. 启用 IP 转发(可选,用于中间人攻击)

如果要实现真正的中间人攻击(MITM),需要在攻击机上启用 IP 转发:

**Windows (管理员权限):**
```cmd
# 启用 IP 转发
netsh interface ipv4 set interface "你的网卡名称" forwarding=enabled

# 或者修改注册表
reg add "HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters" /v IPEnableRouter /t REG_DWORD /d 1 /f
```

**不启用 IP 转发的效果:**
- 目标主机的网络会中断(拒绝服务攻击)
- 目标主机无法访问网关和互联网

**启用 IP 转发的效果:**
- 流量会经过攻击机转发
- 可以使用 Wireshark 等工具抓包分析
- 目标主机网络正常,但流量被监听

## 验证欺骗是否生效

### 在目标主机上检查 ARP 缓存:

**Windows:**
```cmd
arp -a
```

**Linux/Mac:**
```bash
arp -n
```

查看网关 IP 对应的 MAC 地址是否变成了攻击机的 MAC 地址。

### 使用 Wireshark 抓包

1. 在攻击机上启动 Wireshark
2. 选择对应的网卡
3. 过滤器输入: `ip.src == 目标IP or ip.dst == 目标IP`
4. 如果能看到目标主机的流量,说明欺骗成功

## 可能失败的原因

### 1. 目标设备有 ARP 防护
- Windows Defender 或其他安全软件可能检测并阻止
- 解决方案: 在测试环境中关闭安全软件

### 2. 交换机有保护机制
- 动态 ARP 检测 (DAI)
- 端口安全 (Port Security)
- 解决方案: 在自己控制的网络环境中测试

### 3. 静态 ARP 绑定
- 目标主机或网关配置了静态 ARP 表
- 解决方案: 无法绕过,需要其他攻击方式

### 4. 网卡不支持混杂模式
- 某些虚拟机或无线网卡不支持
- 解决方案: 使用有线网卡或更换虚拟机网络模式

## 测试建议

### 安全的测试环境:
1. 使用自己的虚拟机网络
2. 在隔离的测试网络中进行
3. 获得所有设备所有者的授权

### 推荐测试场景:
- 攻击机: Windows 10/11 (物理机或虚拟机)
- 目标机: Windows 10/11 虚拟机
- 网络: VMware NAT 网络或 VirtualBox Host-Only 网络

## 法律警告

⚠️ **重要提示:**
- ARP 欺骗是一种网络攻击行为
- 未经授权对他人网络进行 ARP 欺骗是违法的
- 仅用于授权的安全测试和教育目的
- 在自己的网络环境中进行测试

## 故障排查

### 欺骗包正在发送但无效果:

1. **检查网卡选择**
   - 确保选择了正确的物理网卡
   - 虚拟网卡可能无法正常工作

2. **检查 Npcap 安装**
   - 确保 Npcap 已正确安装
   - 重启计算机后再试

3. **检查防火墙**
   - Windows 防火墙可能阻止
   - 临时关闭防火墙测试

4. **检查目标主机**
   - 在目标主机上运行 `arp -a` 查看 ARP 表
   - 如果网关 MAC 没有变化,说明欺骗未生效

5. **增加发送频率**
   - 将发送间隔改为 500ms 或更短
   - 某些系统会定期刷新 ARP 缓存

## 高级技巧

### 双向 ARP 欺骗
同时欺骗目标主机和网关,实现完整的中间人攻击:
1. 欺骗目标: 告诉目标"网关的 MAC 是我"
2. 欺骗网关: 告诉网关"目标的 MAC 是我"

(当前版本仅支持单向欺骗)

### 配合其他工具
- **Wireshark**: 抓包分析流量
- **Ettercap**: 更强大的 MITM 工具
- **Bettercap**: 现代化的网络攻击框架
